# 行政RFP納品物 検収採点ルーブリック(初稿)

- 対象: 架空の行政RFP(地方公共団体 基幹業務システムのガバメントクラウド(AWS)移行を想定)に対する納品物一式
- 作成日: 2026-06-10
- 位置づけ: 人間レビュー前の初稿。自信のない記述には [要確認] を付している。
- 重要度の定義: **Block** = 検収不可(是正完了まで受領しない) / **Warn** = 条件付き受領(是正計画書の提出を条件とする) / **Info** = 改善提案(受領可)
- 採点方法の定義: **機械** = スクリプト・diff・linter等で機械的に確認可能 / **目視** = 専門家によるレビューが必要 / **機械+目視** = 機械抽出の上で目視判定

## 採点運用ルール

1. **Block項目**が1件でも不合格の場合、検収不可。是正後に当該項目のみ再判定する。
2. **Warn項目**の不合格は、是正計画書(期限・責任者付き)の提出を条件に受領可とする。Warn不合格が全Warn項目の30%を超える場合は検収会議で受領可否を別途協議する [要確認: 閾値は発注者裁量]。
3. **Info項目**は受領可否に影響しないが、次フェーズ調達の評価情報として記録する。
4. 制度・標準(カテゴリC)の判定は**検収実施日時点の現行版**との突合で行う。本ルーブリック作成日(2026-06-10)以降の制度改定は検収時に再確認すること(特にISMAP管理基準の全面改定(2027年3月公表予定)、標準化基本方針の再変更、総務省ガイドラインの改定)。
5. 機械確認可能な項目は採点スクリプトを優先し、目視判定は2名以上のクロスレビューとする。
