## 1. 本書の位置付けと要件確定の方針

1. 本書は、IPA「共通フレーム2013」[^1]の要件定義プロセスに基づき、RFP第4章(機能要件 REQ-001〜028)・第5章(非機能要件 NFR-A01〜F02)を、質問票回答(全9件、令和8年6月10日回答)により確定した**要件ベースライン**である。以後の設計・テストは本書の要件IDを起点に追跡する(KSM-TRM-001)。
2. 要件の**追加・変更・削除はすべて本書2.2「要件変更管理表」に経緯を記録**した。QA回答との対応の詳細は KSM-QAR-001 を参照。
3. 非機能要件は IPA「非機能要求グレード2018」[^2]の体系(可用性/性能・拡張性/運用・保守性/移行性/セキュリティ/システム環境・エコロジー)に対応付けて定量化した。
4. 本書の確定後に生じる要件変更は、課題・リスク管理表に起票の上、影響分析(トレーサビリティマトリクスによる波及確認)を経て両者合意により本書を改版する。

## 3. 業務要件

### 3.1 対象施設・対象業務

RFP第3章のとおり、計34施設(市民体育館1、市民文化会館1、地区公民館12、テニスコート3施設8面、多目的グラウンド4、野球場2、武道館1、その他貸館10)を対象とし、空き状況照会、利用者登録、予約申込(先着・抽選)、利用承認、利用料金算定・収納、減免処理、施設供用管理、統計・実績報告を対象業務とする。現行システム未対応の公民館4館(紙台帳運用)も対象に含める。

### 3.2 主要業務フロー(確定版・概要)

詳細な業務フロー図は P1 期間中の業務部会(週2回・半日枠)で確定し、別冊「業務フロー図」として整備する。確定済みの主要フローの骨子は次のとおり。

| フロー | 流れ(To-Be) | 関連要件 |
|---|---|---|
| F-1 利用者登録 | 住民:オンライン仮申請 → 職員:本人確認(窓口またはオンライン提出書類)→ 本登録 → ID通知。将来:デジタル認証アプリ等連携時はオンライン完結を可能とする拡張性を確保 | REQ-001、003、004 |
| F-2 先着予約 | 空き照会(未ログイン可)→ ログイン → コマ選択(連続・一括・定期)→ 上限チェック → 料金自動算定 → 申込確定 → メール通知 → オンライン決済または窓口納付 | REQ-006、007、009〜012、015、016 |
| F-3 抽選予約 | 申込期間(毎月1〜7日、翌々月分)→ 自動抽選(8日)→ 当落メール通知 → 当選確定(支払期限管理)→ 期限超過は自動取消・繰上げ運用 | REQ-008、012 |
| F-4 減免・収納 | 減免申請(オンライン/窓口代行)→ 所管課承認ワークフロー → 料金再計算 → 収納(オンライン決済/窓口現金/納付書・コンビニ)→ 入金消込 → 財務会計CSV出力 | REQ-015〜020 |
| F-5 施設供用管理 | 所管課:休館日・保守日・優先利用枠設定 → 一般予約制限 → お知らせ掲載 | REQ-022、026 |
| F-6 統計・監査 | 月次・年度の利用率・収納額・減免額集計 → 画面照会・CSV/PDF出力 → 議会・監査対応 | REQ-024、025 |
